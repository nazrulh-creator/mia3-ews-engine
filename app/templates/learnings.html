Key concepts

What MIA3 is

A smoke-detector for the guaranteed book: it watches loans already on the book and flags those likely to slip into Months-in-Arrears 3. (MicroFlex decides new loans; MIA3 monitors existing ones.)

Learn more →

The risk score & four bands

0.5·rank P(MIA3) + 0.3·rank EAD + 0.2·rank Outstanding ratio → Very High, High, Moderate, Low. Probability, impact and leverage in one number.

Learn more →

Confidence & review routing

A 0–100 confidence (five components) decides what a human sees first — borderline cases are held for review.

Learn more →

Governance & audit

Weights, bands, calibration and model changes are dual-controlled and recorded on a hash-chained, tamper-evident trail.

Learn more →

Human-on-the-loop

Precision is very low (~0.1) by design (high recall) — so a flag is a prompt for a person to look, never an automated action against a customer.

Learn more →

LIVE vs TEST

Two ringfenced environments with separate databases and audit chains. Test data never reaches LIVE.

Frequently asked questions

Is a flag a decision?

No. A flag prioritises an account for a human to review. The engine never takes automated action against a customer while precision is being calibrated.

Why do most flags turn out to be false alarms?

The model is deliberately tuned for high recall — to catch most at-risk accounts early. Precision is very low (~0.1 in recent monthly results), so roughly nine in ten flags are false alarms. The confidence gate and the human review queue are how that trade-off is managed in practice.

What's the difference between MIA3 and MicroFlex?

MicroFlex is a gatekeeper — it decides whether to let a new loan through. MIA3 is a smoke detector — it monitors the loans already on the book.

Can the risk weights or band cut-offs be changed?

Yes — internal risk can change them on the Tuning screen, with a re-banding preview and a second approver (dual control). They are governed settings, never code.

Is this scoring the real portfolio yet?

The hosted TEST app runs a synthetic stand-in model on synthetic data. The back-tested XGBoost model drops in via the model registry / MIA3_MODEL_PATH.

Where does the data go?

Synthetic data only in TEST; nothing leaves the engine, and every step is written to the audit trail.
